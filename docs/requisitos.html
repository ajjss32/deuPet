1. Introdução

Este documento tem como objetivo apresentar uma descrição detalhada dos requisitos do aplicativo Deu Pet. Por meio deste, o cliente tomará conhecimento das funcionalidades do projeto e os desenvolvedores terão uma noção de como implementar essas funcionalidades.

1.1 Objetivos

Dentre os objetivos deste documento estão o de listar da forma mais clara possível, os requisitos funcionais, não funcionais e de interface do sistema em questão. Assim, como dito anteriormente, o cliente tomará conhecimento do que o sistema fará e os desenvolvedores de como poderão implementar as funcionalidades.

1.2 Escopo

O presente documento visa ser escrito de forma abrangente e clara para que possa ser utilizado por todos os desenvolvedores, analistas e arquitetos envolvidos e ainda ser compreendido pelo cliente.

1.3 Visão Geral do Documento

A seguir são apresentadas as divisões deste documento e uma rápida descrição de cada seção:

  • Seção 1 - Introdução
  • Seção 2 – Descrição Geral do Sistema: descreve o escopo do sistema e seus usuários de maneira geral.
  • Seção 3 – Requisitos Funcionais: especifica todos os requisitos funcionais planejados para o sistema.
  • Seção 4 – Requisitos Não-Funcionais: especifica todos os requisitos não-funcionais da primeira iteração do sistema.
  • Seção 5 – Requisitos de Interface: especifica todos os requisitos de interface planejados para o sistema.
  • Seção 6 – Matriz de Rastreabilidade
  • Seção 7 - Assinaturas

1.4 Padrões e Convenções

1.4.1 – Identificação dos Requisitos

Para a especificação dos requisitos utilizaremos a seguinte representação:

[TIPODOREQUISITONúmero] Nome

O campo TIPODOREQUISITO poderá ser especificado pelos códigos RF (Requisitos Funcionais), NF (Requisitos Não-Funcionais) ou RI (Requisitos Funcionais de Interface). Já o campo Número será preenchido com um número correspondente à ordem em que os requisitos aparecem no documento.

1.4.2 – Prioridade dos Requisitos

A cada requisito será atribuída uma prioridade. A descrição de cada uma segue abaixo:

  • Essencial: requisito imprescindível. Sem ele, o sistema não funcionará.
  • Importante: requisito que deve ser implementado, mas, se não for, o sistema funcionará do mesmo jeito, mas de maneira insatisfatória.
  • Desejável: requisito que trará um diferencial adicional ao sistema. Por isso, pode ser deixado para ser implementado por último ou em próximas iterações.

2. Descrição Geral do Sistema

A necessidade de um ambiente para auxiliar no processo de adoção de pets de forma online foi necessária atualmente. Há uma grande quantidade de animais para adoção em ONG’s, porém a divulgação desses animais não costuma ser efetiva, dificultando o processo de adoção.

Dessa forma, é importante um sistema que facilite o processo de adoção e, além disso, centralize em um só local diferentes necessidades para os cuidados com os pets.

As funcionalidades do sistema serão mostradas, à medida que os requisitos forem explicados, visto que elas convergem para a ideia do software dita anteriormente.

2.1 Escopo Negativo

Não fazem parte do escopo dessa proposta serviços tais como:

  • Desenvolvimento de funcionalidades extras: só serão implementadas as funcionalidades explicitamente descritas.
  • Controles de segurança sofisticados: a segurança do acesso dos usuários será com senha cadastrada.
  • Comprometimento em disponibilizar o software nem de oferecer treinamento de instalação, configuração, administração e utilização do sistema desenvolvido.

3. Requisitos Funcionais

[RF001] Cadastrar usuário

Requisito básico para permitir que os usuários entrem no sistema como pessoas interessadas em adoção de animais, tendo como entrada seus dados pessoais. Vários usuários poderão se cadastrar.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[RF002] Cadastrar ONG e/ou protetor

Permite que a ONG e/ou protetores possam se cadastrar para usar as funcionalidades do aplicativo.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[RF003] Cadastrar pets

Permite que o usuário divulgue os pets que estão disponíveis para a adoção.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[RF004] Buscar animais disponíveis

O sistema terá um mecanismo de busca que selecionará os animais disponíveis para adoção e retornará essa lista para o usuário interessado.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[RF005] Chat para troca de mensagens

O sistema terá um chat para que os interessados em algum animal demonstrem interesse e conversem com os respectivos cuidadores dos pets para possíveis dúvidas e conversar sobre a adoção.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[RF006] Geração de relatórios das adoções realizadas

O sistema deve gerar relatórios com os animais já adotados para melhor acompanhamento no processo de pós-adoção.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[RF007] Notificações

O sistema deve enviar notificações sobre animais recentemente cadastrados no aplicativo.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[RF008] Login no sistema

Permite que tanto usuários quanto ONGs e/ou protetores forneçam dados de login (e-mail e senha) para acessar o aplicativo.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[RF009] Logout no sistema

Permite que tanto usuários quanto ONGs e/ou protetores possam encerrar suas sessões, perdendo os privilégios de acesso e sendo redirecionados para a página de login.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[RF010] Alteração das informações dos animais

Funcionalidade que possibilita que as ONGs e/ou protetores possam realizar alterações nos dados dos animais disponíveis para a adoção, como imagens e descrições.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[RF011] Consulta de ONGs e/ou protetores

O sistema deverá possibilitar que os usuários consigam acessar informações sobre a ONG e/ou protetor independente que está responsável pelo animal.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[RF012] Lista de favoritos

O usuário poderá criar uma lista com os animais do seu interesse, de forma a possibilitar a análise de qual deles poderá atender melhor as suas expectativas antes de prosseguir com o processo de adoção.

Prioridade:

  • ☐ Essencial
  • ☐ Importante
  • ✓ Desejável

4. Requisitos Não-Funcionais

Esta seção apresenta os requisitos não-funcionais do sistema em detalhes. Todos os requisitos não-funcionais aqui apresentados são essenciais para o bom funcionamento do sistema.

[NF001] Interface gráfica com o usuário

A interface deverá ser amigável, simples e intuitiva.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[NF002] Conexão

O sistema deve funcionar quando o dispositivo tiver acesso à internet.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[NF003] Consistência dos dados

Caso ocorra algum erro no processamento de uma transação, o programa deve retornar para um estado anterior consistente, sem que haja o comprometimento da coerência dos dados armazenados.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[NF004] Testabilidade da aplicação e identificação de falhas

Testes unitários, testes integrados, dentre outros, serão realizados visando garantir o correto funcionamento do aplicativo.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[NF005] Restrições de acesso

Cada cliente utilizará um login e uma senha para poder acessar o sistema. Ou seja, cada ator terá acesso a funcionalidades personalizadas.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[NF006] Suporte para funcionamento em larga escala de usuários

O aplicativo deverá ser desenvolvido de forma que o crescimento exponencial do número de usuários não prejudique o funcionamento e o desempenho do sistema.

Prioridade:

  • ✓ Essencial
  • ☐ Importante
  • ☐ Desejável

[NF007] Hardware

O sistema deve funcionar em smartphones com Android a partir da versão 4.4 e iOS a partir da versão 10.

Prioridade:

  • ☐ Essencial
  • ☐ Importante
  • ✓ Desejável

5. Requisitos de Interface

Esta seção apresenta em detalhes os requisitos de interface do sistema. Especificam como um sistema de software interage com outros sistemas, usuários e hardware. Esses requisitos são fundamentais para garantir que o software possa se comunicar de maneira eficaz e eficiente com outras partes do ambiente de tecnologia ou com os usuários finais.

[RI001] Acessibilidade

O sistema deve funcionar corretamente com leitores de telas e oferecer recursos para ajuste de contraste, tamanho e fonte.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[RI002] Responsividade

O layout do sistema deve se adaptar a diferentes tamanhos de tela.

Prioridade:

  • ☐ Essencial
  • ✓ Importante
  • ☐ Desejável

[RI003] Instruções de uso

A interface deve fornecer instruções claras no primeiro uso para orientar os usuários durante suas interações.

Prioridade:

  • ☐ Essencial
  • ☐ Importante
  • ✓ Desejável
